# 常见算法知识库

本文档用于本地 Algorithm RAG 向量库初始化，按算法主题整理常见模型、适用场景、核心不变式、实现模板和易错点。检索时可以按“题型关键词 + 数据范围 + 操作类型”定位算法，例如“区间修改单点查询”“最大化最小值”“离线三维偏序”“矩阵递推加速”。

## 二分答案

二分答案适合把“求最优值”的问题转化为“判断某个值是否可行”的问题。它常见于最小化最大值、最大化最小值、满足约束的最小成本、达到目标的最短时间等题型。

### 核心条件

必须存在一个单调判定函数 `check(x)`：

- 如果要找“最小可行值”，通常满足：当 `x` 可行时，所有更大的 `x` 也可行；当 `x` 不可行时，所有更小的 `x` 也不可行。
- 如果要找“最大可行值”，通常满足：当 `x` 可行时，所有更小的 `x` 也可行；当 `x` 不可行时，所有更大的 `x` 也不可行。

### 常见步骤

1. 明确答案范围 `[left, right]`，范围必须覆盖真实答案。
2. 编写 `check(mid)`，只判断可行性，不在判定中贪心地提前输出答案。
3. 根据目标收缩区间：
   - 最小可行值：`if check(mid): right = mid else: left = mid + 1`。
   - 最大可行值：`if check(mid): left = mid else: right = mid - 1`，注意上取中位数避免死循环。
4. 循环结束后返回边界值。

### 典型题型

- 船运货物的最低运载能力：给定天数，判断容量是否能在限定天数内运完。
- 分割数组的最小最大段和：判断是否能把数组分成不超过 `k` 段且每段和不超过 `mid`。
- 吃香蕉的最小速度：判断速度 `mid` 是否能在 `h` 小时内完成。
- 最大化最小距离：如放置路由器、牛棚安排，判断相邻距离至少为 `mid` 时最多能放多少个。
- 浮点答案：如几何距离、平均值最大化，通常迭代固定次数或直到误差小于 `eps`。

### 易错点

- `check` 的单调方向没有证明，导致二分无效。
- 边界过小或过大，真实答案不在区间内。
- 整数二分中 `mid = (left + right) // 2` 在求最大可行值时可能死循环，常用 `mid = (left + right + 1) // 2`。
- 浮点二分不应使用整数边界更新方式，通常根据误差终止。

## 前缀和

前缀和用于把多次连续区间求和查询降为常数时间。对于数组 `nums`，定义 `prefix[i + 1] = prefix[i] + nums[i]`，则闭区间 `[l, r]` 的和为 `prefix[r + 1] - prefix[l]`。

### 一维前缀和

适用场景：

- 多次区间和查询。
- 子数组和等于目标值：用哈希表统计历史前缀和出现次数。
- 固定长度窗口统计：长度为 `k` 的区间和为 `prefix[i + k] - prefix[i]`。
- 与差分数组配合做区间更新。

常用思路：

- 若 `prefix[j] - prefix[i] = target`，则需要统计此前有多少个 `prefix[i] = prefix[j] - target`。
- 对非负数组可以用滑动窗口，对含负数数组通常用前缀和 + 哈希表，因为窗口单调性不再成立。

### 二维前缀和

若 `sum[i][j]` 表示矩阵中从 `(0,0)` 到 `(i-1,j-1)` 的矩形和，则：

`sum[i][j] = sum[i-1][j] + sum[i][j-1] - sum[i-1][j-1] + grid[i-1][j-1]`

查询左上角 `(r1,c1)`、右下角 `(r2,c2)` 的矩形和：

`sum[r2+1][c2+1] - sum[r1][c2+1] - sum[r2+1][c1] + sum[r1][c1]`

### 差分与前缀和

差分数组适合“多次区间加，最后一次性求结果”：

- 对区间 `[l,r]` 加 `v`：`diff[l] += v`，`diff[r + 1] -= v`。
- 最终对 `diff` 做前缀和得到每个位置的值。

二维差分可用于矩形区域加值，更新四个角后再做二维前缀还原。

### 易错点

- 0 下标和 1 下标混用导致区间端点偏移。
- 二维前缀和容斥漏减或漏加交集。
- 子数组和问题中必须先统计初始前缀和 `0` 出现一次。

## 树状数组 / Fenwick Tree

树状数组是一种支持前缀统计的数据结构，常用于动态维护前缀和、频次、逆序对、排名和离线统计。它的核心是 `lowbit(x) = x & -x`，表示下标 `x` 管辖区间的长度。

### 基本能力

在 1 下标数组上，树状数组支持：

- 单点加：`add(index, delta)`，复杂度 `O(log n)`。
- 前缀查询：`sum(index)` 查询 `[1,index]` 的和，复杂度 `O(log n)`。
- 区间查询：`range_sum(l,r) = sum(r) - sum(l-1)`。

### 典型模板

```python
class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 1)

    def add(self, i, delta):
        while i <= self.n:
            self.bit[i] += delta
            i += i & -i

    def sum(self, i):
        ans = 0
        while i > 0:
            ans += self.bit[i]
            i -= i & -i
        return ans

    def range_sum(self, l, r):
        return self.sum(r) - self.sum(l - 1)
```

### 常见应用

- 逆序对：从左到右或从右到左扫描，按值离散化后统计此前比当前大或小的数的个数。
- 动态排名：维护每个值出现次数，查询小于等于某值的数量。
- 区间加、单点查：用差分思想，区间 `[l,r]` 加 `v` 等价于 `add(l,v)`、`add(r+1,-v)`，查询点值为前缀和。
- 区间加、区间和：维护两个树状数组 `B1`、`B2`，前缀和为 `x * sum(B1,x) - sum(B2,x)`。
- 二分第 k 小：在频次数组上用树状数组倍增查找最小下标 `pos`，使前缀频次至少为 `k`。

### 与线段树的选择

- 如果只需要前缀可合并信息，如求和、频次、最大前缀中的可逆操作，树状数组更简洁、常数更小。
- 如果需要区间最值、复杂懒标记、多种区间操作组合，线段树更通用。

### 易错点

- 树状数组通常使用 1 下标，离散化后要把排名映射到 `1..m`。
- `add(0, delta)` 会死循环，因此下标不能为 0。
- 查询严格小于当前值时应使用 `sum(rank - 1)`，查询小于等于时使用 `sum(rank)`。

## 矩阵快速幂

矩阵快速幂用于加速线性递推。若状态转移可以写成 `state[n] = T * state[n-1]`，则 `state[n] = T^(n-k) * state[k]`，可以用快速幂在 `O(d^3 log n)` 时间内计算，其中 `d` 是状态维度。

### 适用场景

- 斐波那契数列、爬楼梯、牛繁殖等线性递推。
- 递推式形如 `f(n) = a1*f(n-1) + a2*f(n-2) + ... + ak*f(n-k)`。
- 带常数项的递推：把状态增加一维常数 `1`。
- 图上长度恰好为 `k` 的路径计数：邻接矩阵的 `k` 次幂。
- 自动机 DP 的长串计数：把自动机状态转移矩阵快速幂。

### 斐波那契示例

对 `F(n) = F(n-1) + F(n-2)`，可构造：

```text
[ F(n)   ]   [1 1] [F(n-1)]
[ F(n-1) ] = [1 0] [F(n-2)]
```

因此：

```text
[ F(n)   ]         [F(1)]
[ F(n-1) ] = T^(n-1) * [F(0)]
```

### 通用实现要点

- 矩阵乘法三层循环，通常每次加法后取模，避免溢出。
- 快速幂和整数快速幂类似：当指数当前位为 1 时累乘答案矩阵，然后底数矩阵平方，指数右移。
- 初始答案矩阵应为单位矩阵 `I`，满足 `I * A = A`。

### 带常数项递推

若 `f(n) = 2*f(n-1) + 3`，可扩展状态为 `[f(n), 1]`：

```text
[ f(n) ]   [2 3] [f(n-1)]
[ 1    ] = [0 1] [1     ]
```

这样常数项也能通过矩阵乘法表达。

### 易错点

- 状态向量顺序和转移矩阵行列含义不一致。
- 指数偏移错误，例如从 `state[1]` 推到 `state[n]` 需要 `T^(n-1)`。
- `n` 很小时应直接返回初始值，避免负指数。
- 模数乘法需要在内层循环中处理，Python 可直接大整数但仍建议取模降低开销。

## CDQ 分治

CDQ 分治是一种离线分治技巧，常用于处理“前半部分对后半部分的贡献”，尤其适合偏序统计、动态逆序对、带时间维度的离线查询。它把问题按某一维分治，在合并阶段统计左区间元素对右区间元素的影响。

### 核心思想

对区间 `[l,r]`：

1. 递归处理 `[l,mid]` 内部答案。
2. 递归处理 `[mid+1,r]` 内部答案。
3. 在合并阶段统计左半部分对右半部分的跨区间贡献。
4. 为上层保持某个维度有序，通常需要归并排序。

CDQ 的优势是把一个维度交给递归顺序，另一个维度交给排序，第三个维度交给树状数组或双指针统计。

### 三维偏序示例

问题：给定若干点 `(a,b,c)`，统计每个点有多少个点满足 `a' <= a`、`b' <= b`、`c' <= c`。

常见做法：

1. 先按 `a` 排序，第一维由整体顺序保证。
2. CDQ 按 `a` 的顺序分治，统计左半对右半贡献。
3. 合并时按 `b` 排序扫描，左侧满足 `b' <= b` 的点加入树状数组。
4. 树状数组按 `c` 维护频次，查询 `sum(c)` 得到第三维贡献。

### 动态问题中的时间维度

如果操作序列中有修改和查询，可以把时间作为 CDQ 的递归维度：

- 左半区间的修改会影响右半区间的查询。
- 合并阶段只统计“较早修改”对“较晚查询”的贡献。
- 统计完后必须撤销树状数组中的临时更新，避免污染其他分治层。

### 适用信号

- 题目要求离线处理，查询可以重排或整体读入。
- 存在“时间先后 + 值域大小 + 坐标大小”等多维约束。
- 直接数据结构在线维护困难，但贡献关系满足偏序。
- 复杂度目标常为 `O(n log^2 n)` 或 `O(n log n)`。

### 易错点

- 合并阶段加入树状数组的元素没有撤销。
- 相等元素处理不当，导致 `<=` 和 `<` 混淆；重复点通常需要先合并计数。
- 分治后没有按指定维度归并，导致上层合并复杂度退化或统计错误。
- CDQ 是离线算法，不能直接回答必须实时输出且依赖在线状态的问题。

## 双指针与滑动窗口

双指针常用于有序数组、链表和字符串扫描。滑动窗口是双指针的一种，维护 `[left,right]` 区间，并在右指针扩展、左指针收缩之间保持窗口性质。

### 固定窗口

适合窗口长度固定的问题，例如长度为 `k` 的最大平均值、固定长度子数组最大和。通常先计算第一个窗口，再每次加入右端、移除左端。

### 可变窗口

适合最长或最短满足条件的子数组、子串问题，例如无重复字符最长子串、最小覆盖子串、和至少为目标值的最短子数组。

设计窗口时要明确：

1. 窗口内维护哪些状态，如字符频次、当前和、不同元素个数。
2. 什么时候扩展右端。
3. 什么时候收缩左端。
4. 在扩展前、扩展后、收缩前还是收缩后更新答案。

### 易错点

- 只有当窗口性质具有单调性时，可变窗口才可靠；含负数的和约束通常不能直接滑动窗口。
- 最短窗口通常在满足条件后循环收缩并更新答案。
- 最长窗口通常在不满足条件时收缩，保持合法后更新答案。

## BFS 与 DFS

BFS 使用队列按层遍历，适合无权图最短路、层序遍历、最少步数问题。DFS 使用递归或栈深入搜索，适合连通块、回溯、拓扑相关遍历和树形问题。

### BFS 常见模板

- 初始化队列和 `visited` 集合。
- 每次弹出当前节点。
- 遍历相邻节点，未访问则标记并入队。
- 若需要层数，按队列当前长度分层处理。

### DFS 常见注意点

- 递归终止条件必须清晰。
- `visited` 标记避免重复访问。
- 回溯问题中进入分支前选择，退出分支后撤销选择。
- 大图递归可能栈溢出，可改为显式栈。

### 搜索剪枝

- 可行性剪枝：当前状态已经违反约束，立即返回。
- 最优性剪枝：当前代价已经不可能优于答案，停止扩展。
- 记忆化搜索：对重复子状态缓存结果，本质上接近动态规划。

## 动态规划

动态规划适合具有重叠子问题和最优子结构的问题。核心是定义状态、写出转移、确定初始化和遍历顺序。

### 常见状态设计

- `dp[i]` 表示前 `i` 个元素的最优值、方案数或可行性。
- `dp[i][j]` 表示两个维度共同约束下的结果，如前 `i` 个物品、容量 `j`。
- 区间 DP 中 `dp[l][r]` 表示区间 `[l,r]` 的最优值。
- 树形 DP 中 `dp[u][state]` 表示以节点 `u` 为根的子树在某状态下的结果。
- 状压 DP 中 `dp[mask]` 或 `dp[mask][i]` 表示集合状态和当前位置。

### 背包模型

- 0/1 背包：每个物品最多选一次，容量通常倒序遍历。
- 完全背包：每个物品可选多次，容量通常正序遍历。
- 多重背包：可用二进制拆分或单调队列优化。

### 遍历顺序

遍历顺序必须保证转移依赖的状态已经计算完成。若状态存在环，需要重新建模、拓扑排序或使用最短路算法。

### 易错点

- 状态定义不含足够信息，导致转移依赖未来或历史细节。
- 初始化错误，例如求最小值时未设为无穷大。
- 滚动数组优化时方向错误，把同一轮结果重复使用。

## 贪心

贪心每一步选择局部最优，并要求局部最优能推导出全局最优。使用贪心时需要能说明交换论证、排序依据或不变式。

### 常见贪心模式

- 区间调度：按结束时间排序选择最多不重叠区间。
- 合并区间：按起点排序并维护当前最远右端。
- 跳跃游戏：维护当前能到达的最远位置。
- 分发资源：排序后优先满足最紧约束。
- Huffman 合并：每次合并代价最小的两个元素，使用优先队列。

### 正确性证明思路

- 交换论证：证明任意最优解都能交换成包含当前贪心选择的最优解。
- 反证法：假设不选贪心选择更优，推出矛盾。
- 不变式：每一步后维护“当前选择不劣于任意其他选择”。

贪心代码通常很短，但正确性证明比实现更重要。

## 单调栈与单调队列

单调栈用于寻找每个元素左侧或右侧第一个更大、更小的元素；单调队列用于维护滑动窗口中的最大值或最小值。

### 单调栈应用

- 下一个更大元素。
- 柱状图最大矩形。
- 每日温度。
- 子数组最小值之和：统计每个元素作为最小值的贡献范围。

### 单调队列应用

- 滑动窗口最大值、最小值。
- DP 优化中维护一段候选转移的最优值。
- 有长度限制的前缀和最值查询。

### 易错点

- 处理相等元素时要明确使用 `>`、`>=`、`<`、`<=`，否则贡献区间可能重复或遗漏。
- 单调队列中存下标比存值更安全，因为需要判断元素是否过期。

## 并查集

并查集用于维护动态连通性，支持合并集合和查询两个元素是否在同一集合。路径压缩和按大小或按秩合并后，均摊复杂度接近常数。

### 常见应用

- 无向图连通块数量。
- Kruskal 最小生成树。
- 等式约束满足性。
- 网格连通问题，如岛屿数量。
- 带权并查集：维护节点到根的距离、比例或奇偶关系。

### 易错点

- 初始化时每个元素的父节点应为自己。
- 合并前先找根，不能直接连接原节点。
- 带权并查集在路径压缩时需要同步更新权值。

## 最短路

图的最短路算法应根据边权特征选择。

### 常见选择

- BFS：无权图或每条边权为 1。
- 0-1 BFS：边权只有 0 和 1，使用双端队列。
- Dijkstra：非负边权单源最短路，常用优先队列优化。
- Bellman-Ford / SPFA：可处理负权边，并可检测负环；大数据下 SPFA 可能退化。
- Floyd：多源最短路，复杂度 `O(n^3)`，适合点数较小的图。

### 易错点

- Dijkstra 不能直接处理负权边。
- 优先队列中可能有过期状态，弹出时要判断 `dist != current_dist`。
- 建图时注意有向边和无向边差异。
